## 1. 项目概述

AI Hubs 是一个**新一代智能 Agent 平台**，目标是提供一套"流通且易用"的 AI 框架，解决现有框架配置繁琐、技能下载难、RAG 内存过大、Agent 配置复杂、工作流不全面等问题。

核心能力：
- **快速 API/模型配置**："快速 + 详细"双模式。
- **GitHub 技能市场集成**：内置技能 + 一键从 GitHub 导入开源技能。
- **多层记忆保障防幻觉**：Git 式 commit 回退、记忆图谱索引、高无损 LLM 压缩。
- **多端支持**：CLI + Web + Electron 桌面端 + 后台管理。
- **全功能管理**：用户注册登录、Agent / 任务 / 对话 / 技能 / 数据集管理。
- **内置 IDE**：浏览器内编写、运行、调试代码。
- **UI 简洁不偷懒**：功能完整但界面克制。

### 1.1 设计原则
1. 质量最优先，测试全面不遗漏。
2. 每次改动前提交 Git，保证可追溯。
3. 实时数据优先用工具获取，不依赖训练数据凭空作答。

## 5. 核心模块详解（v4）

### 5.1 核心引擎（`backend/app/core`）
- `agent.py`：LangGraph ReAct 循环。
- `llm.py`：`LLMManager`，统一多 provider；**API Key 从 `data/llm_config.json` 读取**（非环境变量）。支持 openai/deepseek/zhipu/ollama/custom，预设 base_url 见 `PROVIDERS`。
- `orchestrator.py`：8 种编排模式。

### 5.2 认证（`backend/app/security.py`）
- JWT（`python-jose`）。
- **密码哈希用原生 `bcrypt`（`bcrypt.hashpw`/`checkpw`）**，不依赖 `passlib`。原因：`passlib 1.7.4` 与 `bcrypt>=4.1` 不兼容，会导致登录 401。依赖固定 `bcrypt>=4.0.1`（服务器 venv 实际为 5.0.0）。
- 默认管理员 `admin / admin123`，仅在账户不存在时 seed。

### 5.3 数据库（`backend/app/database.py`）
- 启动检测：`backend/config.yaml` 中 `database.mysql.password` 非空 → 用 MySQL（aiomysql）；否则回退 SQLite（`data/ai_hubs.db`）。
- `force_sqlite: true` 可强制 SQLite（桌面端/本地）。

## 6. 配置说明

### 6.1 后端配置 `backend/config.yaml`
| 段 | 关键字段 | 说明 |
|----|----------|------|
| `database` | `force_sqlite` / `mysql.{host,port,user,password,database}` | `force_sqlite:false` 且 `mysql.password` 非空 → MySQL |
| `auth` | `jwt_secret` / `jwt_expire_hours` / `default_admin_username` / `default_admin_password` | 认证与默认管理员 |
| `llm` | `default_provider` / `default_model` | 默认值；实际 key 在 `data/llm_config.json` |
| `email` | `smtp_host` / `sender` / `password` | QQ 邮箱验证码 |

环境变量覆盖：前缀 `AIHUBS_`，双下划线分隔层级，如 `AIHUBS_AUTH__JWT_SECRET`。

### 6.2 ⭐ LLM Key 配置 `data/llm_config.json`（关键）
v4 **不读取** `.env` 的 `DEEPSEEK_API_KEY` 等环境变量。必须写：
```json
{
  "provider": "deepseek",
  "model": "deepseek-chat",
  "api_key": "sk-真实key",
  "base_url": "https://api.deepseek.com/v1",
  "temperature": 0.7,
  "max_tokens": 4096
}
```
- 文件每次对话重新读取，修改**无需重启**。
- 也可通过前端「设置 → 模型 / LLM 配置」写入同一文件。

### 6.3 `.env`（经 systemd EnvironmentFile 注入）
含 `OPENAI_API_KEY` / `DEEPSEEK_API_KEY` / `DASHSCOPE_API_KEY` / `ZHIPU_API_KEY` 等占位符。当前 v4 主要用 `llm_config.json`，`.env` 作为兼容保留，不影响核心功能。

## 10. 部署指南（ECS 8.138.24.27）

### 10.1 架构概览
```
浏览器 ──80/8001──▶ Nginx（8.138.24.27）
                       ├── /            → /var/www/ai_hubs/dist（frontend 构建产物）
                       └── /api /ws /health → 反代 127.0.0.1:8080
                                        ▲
                          systemd 服务 ai_hubs（backend/app/main.py, 8080）
                            ├── MySQL 8.0（ai_hubs）
                            └── data/（llm_config.json / chroma / SQLite 备用）
```

### 10.2 systemd 服务（关键）
`deploy/ai_hubs.service` 必须指向 **v4 入口**：
```ini
WorkingDirectory=/root/ai_hubs/backend
ExecStart=/root/ai_hubs/venv/bin/python -m app.main --host 0.0.0.0 --port 8080
EnvironmentFile=/root/ai_hubs/.env
```
> ⚠️ 切勿指向根目录 `main.py`（v3），否则 API 全部 405。

### 10.3 配置 MySQL
```sql
CREATE DATABASE IF NOT EXISTS ai_hubs CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER IF NOT EXISTS 'ai_hubs'@'localhost' IDENTIFIED BY 'AiHubs@2024!';
GRANT ALL PRIVILEGES ON ai_hubs.* TO 'ai_hubs'@'localhost';
FLUSH PRIVILEGES;
```

### 10.4 服务器本地配置（不进 git，data/ 与 .env 均被 gitignore）
1. `backend/config.yaml`：`force_sqlite: false` + `mysql.password: AiHubs@2024!`。
2. `data/llm_config.json`：写入真实 LLM Key（见 6.2）。
3. `.env`：填入各 provider Key 占位（兼容保留）。

### 10.5 前端构建与上传（本地完成）
服务器内存受限，`npm build` 在本地 Windows：
```powershell
cd frontend; npm install; npm run build
scp -r -o ConnectTimeout=15 "frontend\dist\*" root@8.138.24.27:/var/www/ai_hubs/dist/
ssh root@8.138.24.27 "systemctl reload nginx"
```
> 静态文件必须放 `/var/www/ai_hubs/dist`，**不能**放 `/root/ai_hubs/frontend/dist`（`/root` 权限 700，Nginx 的 www-data 读不到 → 500）。

### 10.6 Nginx 反代
- `location /`：托管 SPA，`try_files $uri /index.html;`（**不要加 `$uri/`**，否则根路径死循环 500）。
- `/api/`、`/ws`、`/health`：反代 `127.0.0.1:8080`；`/api/chat` 关 buffering 支持 SSE。

### 10.7 数据库迁移（v3→v4 一次性）
v4 模型相对 v3 表结构新增列，`create_all` 不会修改已存在表，需手动 `ALTER`：
- `users` 加 `preferences`（JSON，可空）。
- `tasks` 加 `mode` / `think_depth` / `think_visibility`（可空、有默认）；加 `user_id`（NOT NULL 外键 → `users.id`）：先加可空列 → 旧行回填管理员 `id=1` → 改 NOT NULL → 加外键。
- 迁移脚本应使用 `ALTER TABLE ... ADD COLUMN`，并在回填后建立外键约束。

### 10.8 更新流程
```bash
# 本地
git push origin main
cd frontend; npm run build
scp -r "frontend\dist\*" root@8.138.24.27:/var/www/ai_hubs/dist/

# 服务器
ssh root@8.138.24.27 "cd /root/ai_hubs && git pull && systemctl restart ai_hubs"
```

## 12. 常见问题排查

### 12.1 所有 API 返回 405
原因：systemd 跑的是旧 v3 入口（根 `main.py`）。解决：改 `ExecStart` 指向 `backend/app/main.py` 并 `daemon-reload && restart`。

### 12.2 登录返回 401
原因：
- 密码哈希与 v4 不兼容（曾用旧算法或 `passlib` 崩溃）→ 用 v4 的 `hash_password` 重置管理员密码。
- `passlib` 与 `bcrypt` 版本冲突 → 卸载 `passlib`，确认 `security.py` 用原生 `bcrypt`，依赖固定 `bcrypt>=4.0.1`。

### 12.3 对话无响应 / 模型不可用
`data/llm_config.json` 的 `api_key` 为空或错误。填入真实 Key（如 DeepSeek）即可，无需重启。

### 12.4 Nginx 返回 500（静态读不到）
静态根目录设在 `/root/...` 下导致权限不足。改为 `/var/www/ai_hubs/dist`。

### 12.5 根路径 `/` 返回 500 但 `/index.html` 正常
`try_files $uri $uri/ /index.html;` 触发死循环。改为 `try_files $uri /index.html;`。

### 12.6 数据库连不上 / 数据丢失
检查 `backend/config.yaml` 的 `database` 段：`force_sqlite` 是否为 `false`、`mysql.password` 是否填写；MySQL 是否运行、用户权限是否到位。未配置则回退 SQLite（重启不丢，但非 MySQL）。
